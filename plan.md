# YOLOv8 优化策略调优组项目计划

## 1. 个人任务定位

本仓库用于完成皇甫泊宁在小组项目中的个人部分：**YOLOv8 优化策略调优实验**。

小组统一使用 SKU-110K 数据集进行超市货架密集商品检测实验。本人的变量设置为：在 YOLOv8n 预训练权重基础上，将默认优化策略中的 SGD 替换为 AdamW，观察优化器变化对模型微调效果、收敛过程和真实场景泛化能力的影响。

## 2. 实验目标

1. 使用 SKU-110K 数据集完成 YOLOv8n 的目标检测微调。
2. 保持模型结构和主要训练设置与 Baseline 组尽量一致，仅重点改变优化器为 AdamW。
3. 记录 AdamW 微调过程中的关键参数、Loss 曲线和训练日志。
4. 对比微调前后模型在测试集上的检测性能变化。
5. 使用 5 张非同源超市货架图片进行额外验证，分析模型在真实场景中的泛化效果。

## 3. 数据与格式准备

当前实现：已新增 `scripts/prepare_sku110k.py`、`data/README.md` 和 `data/sku110k.yaml`，用于将 SKU-110K CSV 标注转换为 Ultralytics YOLO 检测数据集格式。当前本地已完成 `data/raw/SKU110K_fixed` 解压，并生成 `data/processed/sku110k` 的 train、val、test 三个划分。

### 3.1 数据集

- 数据集：SKU-110K
- 场景：超市货架密集商品检测
- 标注：商品边界框标注
- 任务类型：单类或少类密集目标检测，具体类别设置以数据集标注文件清洗结果为准

### 3.2 数据预处理计划

1. 下载并整理 SKU-110K 原始图像与标注文件。
2. 检查标注文件中的异常项，例如空标注、越界框、宽高为 0 的框、损坏图片。
3. 将原始标注转换为 YOLO 格式：`class x_center y_center width height`。
4. 坐标统一转换为相对图片宽高的归一化数值。
5. 按小组统一规则划分训练集、验证集和测试集。
6. 编写或整理 `data.yaml`，供 Ultralytics YOLO 训练与验证调用。

## 4. 模型与实验设置

当前实现：已新增 `configs/adamw.yaml` 和 `scripts/train_adamw.py`，用于启动 YOLOv8n + AdamW 微调实验。`yolov8n.pt` 已下载到 `models/yolov8n.pt`，实验配置默认使用该本地权重路径。

### 4.1 基础模型

- 模型版本：YOLOv8
- 预训练权重：`yolov8n.pt`
- 实验定位：优化器策略调优，不改变模型规模

### 4.2 核心变量

- Baseline 参考：`yolov8n.pt` + 默认超参数
- 本人实验：`yolov8n.pt` + `optimizer=AdamW`
- 其余训练参数应尽量与 Baseline 组保持一致，便于横向比较

### 4.3 需要记录的关键参数

- 训练轮数 `epochs`
- 输入尺寸 `imgsz`
- 批大小 `batch`
- 优化器 `optimizer`
- 初始学习率 `lr0`
- 权重衰减 `weight_decay`
- 数据增强设置
- 训练设备与显存环境
- 随机种子或可复现实验设置

## 5. 量化评估计划

### 5.1 微调前评估

使用原始 `yolov8n.pt` 在 SKU-110K 测试集上进行验证，记录未微调模型的初始检测性能。

重点记录指标：

- Precision
- Recall
- mAP@0.5
- mAP@0.5:0.95
- 推理结果中的典型漏检和误检现象

### 5.2 微调后评估

使用 AdamW 微调后的最佳权重在同一测试集上评估，记录同样指标并与微调前结果对比。

重点分析：

- mAP 是否提升
- Precision 与 Recall 的变化方向
- 密集商品场景下漏检是否减少
- 是否出现更高误检或框定位不稳定问题
- Loss 曲线是否收敛平稳，是否存在过拟合迹象

## 6. 额外五张图片验证计划

### 6.1 图片来源

自行从网络收集 5 张非 SKU-110K 来源的真实超市货架图片，尽量覆盖以下情况：

- 密集排列商品
- 多层货架
- 商品尺度变化明显
- 遮挡或重叠较多
- 拍摄角度与光照条件不同

### 6.2 对比方式

对每张图片分别使用以下模型推理：

1. 微调前模型：`yolov8n.pt`
2. 微调后模型：AdamW 微调得到的最佳权重

### 6.3 分析重点

- 检测框数量变化
- 置信度变化
- 漏检情况
- 误检情况
- 密集重叠区域的框定位质量
- 微调后模型对真实零售货架图片的泛化能力

## 8. 阶段安排

### 阶段一：环境与数据准备

- 安装并确认 Ultralytics YOLO 可运行。
- 下载 SKU-110K 数据集。
- 完成标注清洗、格式转换和数据划分。
- 准备 `data.yaml` 并验证数据路径正确。

### 阶段二：微调前基准评估

- 使用 `yolov8n.pt` 在测试集上评估。
- 保存指标结果、预测示例图和验证日志。
- 记录初始模型在 SKU-110K 场景下的主要问题。

### 阶段三：AdamW 微调实验

- 使用 `yolov8n.pt` 作为预训练权重。
- 将优化器设置为 AdamW。
- 训练过程中保存 Loss 曲线、关键参数和最佳权重。
- 如训练异常，优先检查学习率、batch size、数据标注质量和显存限制。

### 阶段四：微调后性能验证

- 使用最佳权重在测试集上评估。
- 与微调前模型进行指标对比。
- 整理 Precision、Recall、mAP@0.5、mAP@0.5:0.95 等结果。

### 阶段五：额外图片泛化验证

- 收集 5 张非同源超市货架图片。
- 分别使用微调前后模型推理。
- 保存可视化结果。
- 针对每张图片分析检测质量变化。

### 阶段六：个人报告整理

- 汇总微调过程设置。
- 汇总测试集性能对比。
- 汇总额外图片推理结果。
- 重点讨论 AdamW 相比默认优化策略可能带来的收敛与泛化差异。

## 10. 风险与注意事项

- SKU-110K 数据量较大，训练时间和显存占用需要提前评估。
- 密集目标检测对标注转换质量非常敏感，预处理阶段必须重点检查坐标归一化是否正确。
- AdamW 的效果受学习率和权重衰减影响明显，若直接使用默认设置效果不佳，需要在报告中客观分析原因。
- 与 Baseline 组对比时，应确保除优化器外的关键变量尽量一致，否则结论不够严谨。
- 额外图片不应来自训练集或测试集，避免泛化验证失真。
